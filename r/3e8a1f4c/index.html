Hardware De-Risk Roadmap

A phased strategy for validating the physics and manufacturing path of the NetAnywhere node — from breadboard through sellable product to production-grade wafer-level packaging.

Phase 0
ESP32-C6 Breadboard
Radio Baseline
A cluster of dev boards, wires, and breakout modules spread across a desk. Each solar panel is roughly credit card sized — and each node needs one. Nothing is integrated; components are connected by jumper wires. This is a functional proof-of-concept, not a product form factor.
Done
Cost ~$250
Started Mar 30, 2026
Target completion ~Apr 15, 2026

Does the solar harvest → store → relay loop close? What range and power numbers do we actually see on the bench and outdoors?

Demonstrated on hardware
  • BLE + WiFi simultaneous operation impossible on ESP32-C6 — WPA2 handshake fails under active BLE mesh traffic; time-multiplex workaround confirmed functional measuredMar 30, 2026
  • 4-node BLE mesh relay proven indoors: shielded node reached over 18 ft + 90° turn via relay measuredMar 31, 2026
  • Outdoor range at 56th & 1st Ave, NYC: 35 m at +3 dBm, 88 m at +20 dBm (1M PHY) measuredMar 31, 2026
  • Power subsystem wired: BQ25570 + LM3671 + Voltaic P121 + supercap — all components confirmed functional individually measuredApr 1–4, 2026
  • Indoor bench test: VSTOR sags to 2.73 V under DevKitC load — root cause identified as DevKit board overhead (~70 mW USB bridge chip), not a solar panel or harvester fault measuredApr 4, 2026
Estimated from datasheets (not yet bench-measured)
  • BLE duty-cycled idle: ~0.42 mW target; continuous ESP-BLE-MESH: ~20 mW (ESP32-C6 datasheet)
  • WiFi soft-AP draw: ~200–400 mW burst; 10 F supercap sustains ~60–90 s window (calculated)
  • Bare ESP32-C6 module draw (no USB bridge): ~20–30 mW active — well within P121 indoor harvest window
In progress
  • Seeed XIAO ESP32-C6 (Pre-Soldered) ordered — bare-module board with no USB bridge chip; directly measures ESP32-C6 silicon draw vs. DevKit overhead
  • Phase 7a: repeat indoor bench test with XIAO to confirm DevKit overhead was the cause of VSTOR sag
  • Phase 7b / Step 4 outdoor: take to direct sunlight — confirm VSTOR holds above 3.5 V under both DevKitC and XIAO load
  • Phase 7c: solar-only relay chain with XIAO nodes — 3-node mesh, no USB, outdoor

"Range is proven on hardware in the field. The power subsystem is wired and working — we traced the indoor deficit to 70 mW of dev kit overhead, not a physics problem. The ESP32-C6 established the radio baseline; the Nordic dual-chip breadboard is next."

Phase 1
Nordic nRF5340+nRF7002 Breadboard
End-to-End Relay Demo
Three breadboard nodes. Two phones — one connected to each end node over Wi-Fi. A relay node in the middle. The phones send a message. It travels Phone → Wi-Fi → 802.15.4 → relay → 802.15.4 → Wi-Fi → Phone. When it arrives, the architecture is proven.
In Progress
Started Apr 18, 2026
Duration 2–4 weeks

Do two Wi-Fi clients communicate end-to-end through an 802.15.4 relay on real hardware?

Demonstrated on hardware
  • All hardware arrived; full dev environment operational (nRF Connect SDK v2.9.0, west, Zephyr SDK, J-Link, usbipd WSL2 passthrough) confirmedApr 18, 2026
  • Wi-Fi 6 (802.11ax) connection validated: 5 GHz, WPA2-PSK, RSSI −22 dBm, DHCP assigned — clean boot on nRF5340-DK + nRF7002 EK measuredApr 18, 2026
  • Concurrent Wi-Fi + 802.15.4 (Thread) coexistence closed — G-N1a: Wi-Fi ~4.78 Mbps and Thread ~63 kbps running simultaneously on the same node measuredApr 18, 2026
In progress
  • G-N1b: three-node relay field demo — two laptops, three USB-powered nodes, static Thread credentials baked into firmware. Edge nodes serve a SoftAP web UI with message send + status pill; relay node bridges them over 802.15.4. Test closes when a message traverses the full path and the relay survives a power-cycle mid-test.
Setup
  • 3× nRF5340-DK (Nordic dev kit, nRF5340, 802.15.4 backbone); 2× nRF7002 EK (Wi-Fi 6 shield, one per edge node)
  • Edge nodes run Wi-Fi SOFTAP + 802.15.4 simultaneously — the combination rated X (not supported) on the ESP32-C6
  • Relay node runs 802.15.4 only; bridges the two edges
  • Test passes when a message sent from Laptop 1 arrives at Laptop 2 with the relay node as the only path

"Phase 0 proved the range and identified the power problem — dev board overhead, not the physics. Phase 1 proves the product works: two laptops, out of range of each other, communicating through a relay node over radio. No infrastructure. Coexistence is closed on real hardware. The relay demo is next."

2
Phase 2
Nordic nRF5340 Surface-Mount (SMT) Board
Solar-Powered Node
A green circuit board about 2 inches square — larger than a Triscuit, smaller than a playing card — with nRF5340 only (802.15.4 mesh backbone), a solar cell, and a supercap. No Wi-Fi radio on the node — laptop/phone access stays on the carried gateway path, same as the field architecture. Inductor-less, PMIC-less direct-drive power (no BQ25570). No dev board overhead — real silicon and a minimal 2-layer layout. This is the first board where solar validation is meaningful.
Planned
Estimated cost ~$200–500
Duration ~5 weeks (design + fab)

Does the Nordic nRF5340-only node run on solar with direct-drive power? This is the first board with no dev kit overhead — real chip, real mesh radio, real power budget — and no Wi-Fi block to mask current draw.

Approach
  • ~40–50 mm square, standard 2-layer FR4 (JLCPCB-friendly), nRF5340 in QFN only — no nRF7002, no extra LEDs, no USB bridge
  • Direct-drive solar-to-storage path: solar cell + supercap, inductor-less and PMIC-less — BOM and assembly simplified vs. a harvester-PMIC stack
  • 802.15.4 (Thread or other) only on-board; Wi-Fi for setup or backhaul stays off this unit (gateway / dongle path)
  • First real solar validation: does the node stay alive on ambient light with no USB?
  • Fully probeable — all test points accessible for debugging before committing to HDI miniaturization
  • Fast-turn fab: ~$10/board, ~1 week at vendors such as PCBWay

"Phase 1 proved coexistence and relay on Nordic dev kits. Phase 2 strips the Wi-Fi sidecar and proves the field-node power budget on a real SMT board — no dev kit overhead, no wasted current. If the nRF5340-only node runs on solar here, the physics are closed. Miniaturization is next."

3
Phase 3
Nordic nRF5340+nRF7002 Miniaturized Circuit Board
First Deployable Product
A rigid square roughly the size of a Triscuit (~1.5 × 1.5 inches), about as thick as a stack of 3–4 credit cards. nRF5340 + nRF7002 in WLCSP/QFN packages, solar film bonded to both faces. You can hold one between two fingers. This is the first deployable product.
Planned
Estimated cost ~$2–5k
Duration 6–10 weeks
Est. completion Jul–Aug 2026

Does the power budget close when the node fits in the palm of your hand? Can it ship to defense and enterprise customers?

Architecture
  • 25 × 25 mm HDI PCB; nRF5340 WLCSP + nRF7002 QFN — separate RF paths, hardware PTA
  • PMIC: BQ25570 DSBGA or e-peas AEM10941; supercap: SMD EDLC, 0.9 mm profile
  • Thin-film PV bonded to top and bottom faces; total thickness ~3.5–4.2 mm
  • Assembly: standard SMT pick-and-place at MacroFab or Screaming Circuits

"We have a functional node at target footprint that you can hold in your hand. The power budget closes. This ships to DoD programs and enterprise customers today. Shrinking from 4 mm to 1 mm is a manufacturing transition, not a physics question — and that is what Phase 4 funds."

4
Phase 4
Bare-Die-on-Flex
Scatter Form Factor
Shrinks from the Triscuit to a Wheat Thin (1 inch × 1 inch) — and from a stack of credit cards down to one. Flexible substrate instead of rigid PCB. Solar film bonded to both faces. The unit bends slightly. At this thickness and weight, it can tumble and drift on the wind when dropped from altitude — the first unit truly suited to scatter deployment.
Planned
Estimated one-time setup cost $30–50k
Duration 3–6 months
Est. completion Q4 2026 – Q1 2027

Can we hit the ~1 mm Z-axis profile required for aerodynamic scatter deployment?

Architecture
  • 50 µm polyimide flex substrate with RDL antenna; bare dies via Micross / SemiDice
  • Energy storage: CAP-XX ultra-thin prismatic supercap pouch (~0.4 mm)
  • Passives: Silicon Capacitors (SiCaps) or 01005 for zero Z-height impact
  • Assembly at OSAT: QP Technologies or Promex Industries
  • Total target thickness: ~0.95–1.15 mm

"Phase 3 proved the physics. Phase 4 proves the form factor. The supply chain here maps directly to the high-volume reel-to-reel manufacturing that hits $1/node."

5
Phase 5
FOWLP Multi-Die Package
Volume Production
Visually identical to Phase 4 — a Wheat Thin, credit card thin — but now a single hermetically sealed package with no visible components, seams, or connectors. Both faces are photovoltaic. The only external surface is SiO₂ passivation. It looks and feels like a smooth, matte ceramic tile. Drop thousands from an aircraft; they self-organize into a mesh on landing. This is the production unit.
Planned
Estimated one-time setup cost $500k – $2M+
Unit cost at volume <$1 / node
Duration 12–24 months
Est. completion 2028+ (funding-dependent)

Can we achieve sub-$1/node unit economics at scale with full wafer-level integration?

Architecture
  • COTS radio dies (nRF5340 + nRF7002 bare die) + custom sensor/PMIC dies embedded in mold compound
  • On-package RDL routing and 2.4 GHz RDL antenna; no flex PCB
  • Thin-film PV bonded at wafer scale to top and bottom; SiO₂ hermetic passivation
  • Prototype partners: IMEC, CEA-Leti, IHP Microelectronics; production scale via ASE Group

"Phases 3 and 4 retired the technical risks. Phase 5 is a pure capital-scaling play — same proven silicon in a wafer-level package that hits <$1/node. This is the inflection point where NetAnywhere becomes a deployable macro-environmental platform."
